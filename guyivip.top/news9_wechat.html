OpenAI Codex CLI：终端里的 AI 编程助手

Seek4nothing · 2025年4月20日

📌 编者按

作为一个习惯使用终端工作的原始人，我一直觉得 IDE 里的 AI 助手虽然方便，但总少了点"命令行战士"的味道。

直到 OpenAI 推出了 Codex CLI——一个完全运行在终端里的 AI 编程助手，我才找到了感觉。它就像一个坐在你身边的结对编程伙伴，你用自然语言描述需求，它直接帮你改代码、跑测试。

今天就来详细介绍一下这个工具，以及如何在日常开发中使用它。

🌟 项目简介

OpenAI Codex CLI 是 OpenAI 推出的轻量级终端编程助手，用 Rust 编写，直接在你的命令行中运行。它能理解你的代码库、执行代码修改、运行测试，就像一个坐在你身边的结对编程伙伴。

💡 核心特点

1. 终端原生

无需打开 IDE，在终端里就能完成所有工作。对于习惯命令行的开发者来说，这是最自然的工作方式。

2. 多模态输入

支持文本、截图、图表等多种输入方式。你可以直接把错误截图丢给它，让它帮你分析问题。

3. 安全沙箱

在 macOS 上使用 Apple Seatbelt 隔离，在 Linux 上使用 seccomp 限制，确保 AI 的操作不会危害你的系统。

4. 代码库感知

自动读取你的项目文件，理解上下文。不需要手动复制粘贴代码，它会自己找到相关文件。

📦 安装方法

# 方法一：npm 安装（推荐）

npm install -g @openai/codex

# 方法二：Homebrew（macOS）

brew install codex

# 方法三：源码编译

git clone https://github.com/openai/codex.git

cd codex && cargo install --path .

⚙️ 配置使用

# 1. 设置 API Key

export OPENAI_API_KEY="your-api-key"

# 2. 进入你的项目目录

cd your-project

# 3. 启动 Codex

codex

# 或者直接给指令

codex "帮我实现用户登录功能"

💡 使用示例

示例 1：修复 Bug

$ codex "运行测试，看看哪些失败了，然后修复它们"

[Codex] 正在运行 pytest...
[Codex] 发现 3 个测试失败
[Codex] 分析失败原因...
[Codex] 已修复 auth.py 第 42 行的类型错误
[Codex] 重新运行测试... 全部通过 ✓

示例 2：代码重构

$ codex "把 utils.py 里的重复代码提取成公共函数"

[Codex] 正在分析代码结构...
[Codex] 发现 3 处重复的字符串处理逻辑
[Codex] 创建通用函数 sanitize_string()
[Codex] 更新 5 个文件的调用点

💡 提示：Codex CLI 支持多种审批模式：
• suggest：只提供建议，需要手动确认
• auto-edit：自动编辑文件，但需要确认执行命令
• full-auto：完全自动（谨慎使用）

📊 适用场景

  • 快速原型开发：描述需求，快速生成代码
  • Bug 修复：报错信息丢给它，自动定位修复
  • 代码重构：批量优化代码结构
  • 测试编写：自动为代码生成测试用例
  • 文档生成：根据代码生成 API 文档
